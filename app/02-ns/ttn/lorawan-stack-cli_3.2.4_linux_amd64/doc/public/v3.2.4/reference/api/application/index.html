Application APIs

The ApplicationRegistry service

Method ApplicationRegistry.Create

Create a new application. This also sets the given organization or user as first collaborator with all possible rights.

Request: CreateApplicationRequest
Response: Application

POST /users/{collaborator.user_ids.user_id}/applications

POST /organizations/{collaborator.organization_ids.organization_id}/applications

Method ApplicationRegistry.Get

Get the application with the given identifiers, selecting the fields given by the field mask. The method may return more or less fields, depending on the rights of the caller.

Request: GetApplicationRequest
Response: Application

GET /applications/{application_ids.application_id}

Method ApplicationRegistry.List

List applications. See request message for details.

Request: ListApplicationsRequest
Response: Applications

GET /applications

GET /users/{collaborator.user_ids.user_id}/applications

GET /organizations/{collaborator.organization_ids.organization_id}/applications

Method ApplicationRegistry.Update

Request: UpdateApplicationRequest
Response: Application

PUT /applications/{application.ids.application_id}

Method ApplicationRegistry.Delete

Request: ApplicationIdentifiers
Response: google.protobuf.Empty

DELETE /applications/{application_id}

The ApplicationAccess service

Method ApplicationAccess.ListRights

Request: ApplicationIdentifiers
Response: Rights

GET /applications/{application_id}/rights

Method ApplicationAccess.CreateAPIKey

Request: CreateApplicationAPIKeyRequest
Response: APIKey

POST /applications/{application_ids.application_id}/api-keys

Method ApplicationAccess.ListAPIKeys

Request: ListApplicationAPIKeysRequest
Response: APIKeys

GET /applications/{application_ids.application_id}/api-keys

Method ApplicationAccess.GetAPIKey

Request: GetApplicationAPIKeyRequest
Response: APIKey

GET /applications/{application_ids.application_id}/api-keys/{key_id}

Method ApplicationAccess.UpdateAPIKey

Update the rights of an existing application API key. To generate an API key, the CreateAPIKey should be used. To delete an API key, update it with zero rights. It is required for the caller to have all assigned or/and removed rights.

Request: UpdateApplicationAPIKeyRequest
Response: APIKey

PUT /applications/{application_ids.application_id}/api-keys/{api_key.id}

Method ApplicationAccess.GetCollaborator

Get the rights of a collaborator (member) of the application. Pseudo-rights in the response (such as the “_ALL” right) are not expanded.

Request: GetApplicationCollaboratorRequest
Response: GetCollaboratorResponse

GET /applications/{application_ids.application_id}/collaborator

GET /applications/{application_ids.application_id}/collaborator/user/{collaborator.user_ids.user_id}

GET /applications/{application_ids.application_id}/collaborator/organization/{collaborator.organization_ids.organization_id}

Method ApplicationAccess.SetCollaborator

Set the rights of a collaborator (member) on the application. It is required for the caller to have all assigned or/and removed rights. Setting a collaborator without rights, removes them.

Request: SetApplicationCollaboratorRequest
Response: google.protobuf.Empty

PUT /applications/{application_ids.application_id}/collaborators

Method ApplicationAccess.ListCollaborators

Request: ListApplicationCollaboratorsRequest
Response: Collaborators

GET /applications/{application_ids.application_id}/collaborators
